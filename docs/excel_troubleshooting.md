# Excel Companion Troubleshooting

## Check Log First

The companion writes user-friendly warnings and errors to the `Log` sheet. If `debug_mode` is `FALSE`, Excel users see friendly errors while technical details stay out of the foreground.

Set `debug_mode` to `TRUE` to include Python traceback details in `Log`.

## Common Issues

Missing data sheet: confirm `data_sheet` matches the sheet name exactly.

Missing range: leave `data_range` blank to use the used range, or provide an A1-style range such as `A1:H25`.

Missing columns: confirm `x_column`, `y_column`, `denominator_column` and `expected_column` match the headers in `Data`.

Invalid chart type: use one of `run`, `i`, `mr`, `xmr`, `c`, `p`, `u`, `xbar`, `s`, `g`, `t`, `p_prime`, `u_prime` or `pareto`.

xlwings cannot find Python: configure the xlwings add-in to point to the same Python environment where pyqicharts is installed.

PowerPoint export fails: install the reporting dependencies with `pip install pyqicharts[excel]`.

## Known Limitations

The supplied `.xlsm` template is an xlwings-ready workbook shell. It does not contain a signed VBA project generated by Python. Add signed VBA/ribbon deployment through your organisation's normal Excel governance process.
